🔧 MODO REAL: EL LEGADO DE LOS 16 BITS

El Modo Real es el modo de operación original de los procesadores Intel 8086 y 8088. Es el estado en el que una computadora con arquitectura x86 inicia. Este modo se caracteriza por su simplicidad, pero también por sus significativas limitaciones.

Características Principales:

Espacio de Direccionamiento de 1 MB:

Utiliza un bus de direcciones de 20 bits, lo que le permite acceder a un máximo de 1 megabyte (MB) de memoria RAM.

[ DIAGRAMA DE MEMORIA - MODO REAL ]

1 MB MÁXIMO (2²⁰ bytes)

Bus de direcciones: 20 bits

  • Memoria Segmentada: El direccionamiento se realiza a través de un esquema de segmentos y desplazamientos. Una dirección de memoria se forma a partir de una dirección de segmento de 16 bits y un desplazamiento (offset) también de 16 bits. La dirección física se calcula desplazando la dirección del segmento 4 bits a la izquierda y sumándole el desplazamiento.
  • Falta de Protección de Memoria: En este modo, no existen mecanismos de hardware para proteger la memoria. Un programa puede acceder y modificar cualquier parte de la memoria, incluyendo el código del sistema operativo u otros programas, lo que lo hace muy vulnerable a errores y a software malicioso.
  • Monotarea: No ofrece soporte a nivel de hardware para la multitarea. Solo un programa se ejecuta a la vez, con acceso directo y sin restricciones a todo el hardware del sistema y a las rutinas del BIOS.

⚠ LIMITACIÓN CRÍTICA:

Sin protección de memoria, cualquier error de programación puede causar que el sistema completo colapse. Los programas tienen acceso ilimitado a todo el hardware.

🛡️ MODO PROTEGIDO: LA ERA DE LA MULTITAREA Y LA SEGURIDAD

Con la introducción del procesador Intel 80286 y su consolidación con el 80386, surgió el Modo Protegido. Este modo fue diseñado para superar las limitaciones del Modo Real y es la base sobre la que operan todos los sistemas operativos modernos de 32 y 64 bits.

Características Principales:

Acceso a Memoria Extendida:

Permite el acceso a la memoria más allá del límite de 1 MB del Modo Real. Con un bus de direcciones de 32 bits (en el 80386 y posteriores), puede direccionar hasta 4 gigabytes (GB) de memoria física.

[ DIAGRAMA DE MEMORIA - MODO PROTEGIDO ]

4 GB MÁXIMO (2³² bytes)

Bus de direcciones: 32 bits (80386+)

  • Protección de Memoria: Introduce un sofisticado mecanismo de protección que impide que un programa interfiera con la memoria de otro o con la del sistema operativo. Esto se logra a través de descriptores de segmento y niveles de privilegio (anillos de protección), donde el sistema operativo se ejecuta en el nivel más privilegiado (anillo 0) y las aplicaciones en niveles menos privilegiados.
  • Soporte para Multitarea: Ofrece soporte a nivel de hardware para la multitarea apropiativa, permitiendo que el sistema operativo gestione y cambie entre múltiples programas en ejecución de manera eficiente y segura.
  • Memoria Virtual: Permite el uso de memoria virtual, una técnica que utiliza el almacenamiento secundario (como un disco duro) para simular una cantidad de memoria RAM mucho mayor a la que está físicamente disponible.
  • Nuevo Modelo de Direccionamiento: En lugar de que los registros de segmento contengan directamente la dirección del segmento, ahora contienen un "selector" que apunta a una entrada en una tabla de descriptores (GDT - Tabla de Descriptores Globales o LDT - Tabla de Descriptores Locales). Este descriptor contiene la dirección base real del segmento, su tamaño y los permisos de acceso.

⚠ ANILLOS DE PROTECCIÓN:

Ring 0 (Kernel) → Ring 1 (Drivers) → Ring 2 (Drivers) → Ring 3 (Aplicaciones)

Cada anillo tiene diferentes niveles de acceso a los recursos del sistema.

🖥️ MODO VIRTUAL 8086: COMPATIBILIDAD CON EL PASADO

A pesar de las ventajas del Modo Protegido, existía la necesidad de ejecutar el vasto software existente diseñado para MS-DOS y que operaba en Modo Real. Para solucionar esto, el procesador Intel 80386 introdujo el Modo Virtual 8086 (VM86).

Características Principales:

Máquina Virtual para el Modo Real:

El Modo Virtual 8086 no es un modo de operación independiente, sino una subtarea dentro del Modo Protegido. Su función es crear un entorno de máquina virtual que simula el Modo Real.

  • Ejecución de Programas de 16 Bits: Permite que los programas de 16 bits, escritos para el Modo Real, se ejecuten dentro de un sistema operativo moderno que opera en Modo Protegido.
  • Aislamiento y Protección: Aunque el programa de 16 bits cree que tiene el control total del sistema (como en el Modo Real), en realidad está siendo supervisado por el sistema operativo. Las acciones que podrían comprometer la estabilidad del sistema (como el acceso directo a cierto hardware) son interceptadas y gestionadas por el sistema operativo, manteniendo así la protección y la multitarea.
  • Multitarea de Aplicaciones de Modo Real: Un sistema operativo en Modo Protegido puede ejecutar múltiples tareas en Modo Virtual 8086 simultáneamente, aislando cada una de ellas y del resto del sistema.

⚠ COMPATIBILIDAD:

Este modo permitió que sistemas operativos como Windows 95/98 ejecutaran aplicaciones MS-DOS mientras mantenían la estabilidad y seguridad del sistema moderno.

📊 CONCLUSIÓN

La evolución del direccionamiento en la arquitectura x86 es un reflejo de la historia de la computación personal: desde el simple y directo Modo Real, pasando por el robusto y seguro Modo Protegido que habilitó los sistemas operativos modernos, hasta el ingenioso Modo Virtual 8086 que aseguró la compatibilidad con el software legado.
